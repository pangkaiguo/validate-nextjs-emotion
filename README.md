# Validate Next.js + Emotion

A validation project to test Emotion CSS-in-JS support in Next.js (App Router).

## 🧪 Test Coverage

This project validates the following Emotion features in Next.js:

| # | Feature | Status |
|---|---------|--------|
| 1 | `css` prop - Object syntax | ✅ |
| 2 | `css` prop - Template literal syntax | ✅ |
| 3 | `styled` components | ✅ |
| 4 | Styled component composition | ✅ |
| 5 | Styled components with props | ✅ |
| 6 | Button states (:hover, :active, :disabled) | ✅ |
| 7 | `keyframes` animation | ✅ |
| 8 | Style composition (`css` \`\` composition) | ✅ |
| 9 | Nested selectors | ✅ |
| 10 | Media queries | ✅ |
| 11 | `Global` styles | ✅ |
| 12 | SSR - Server-side rendering with Emotion | ✅ |
| 13 | TypeScript support with `css` prop | ✅ |

## 🚀 Getting Started

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) to view the validation page.

## 🏗️ Build

```bash
npm run build
npm start
```

## 📦 Tech Stack

- **Next.js 16** - App Router
- **React 19**
- **Emotion 11** - `@emotion/react`, `@emotion/styled`, `@emotion/cache`
- **TypeScript**
